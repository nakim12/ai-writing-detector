# AI Writing Detector

A PSTAT 134 group project that builds NLP classifiers to predict whether a piece of writing was written by a human or generated by AI.

**Authors:** Priyanka Durai, Lindsay Der, Dylan Cino, Victoria Hamman, Nathan Kim

## Project Overview

We use the [Human vs AI Generated Essays](https://www.kaggle.com/datasets/navjotkaushal/human-vs-ai-generated-essays) dataset from Kaggle (2,750 essays, perfectly balanced between human and AI). The goal is primarily **predictive** — given a new essay, predict whether it is AI-generated — with a secondary **descriptive** goal of identifying linguistic features that distinguish AI text from human text.

See [`Project-Memo.qmd`](Project-Memo.qmd) for the full project plan, including the evaluation strategy.

## Getting Started

### 1. Clone and create a virtual environment

```bash
git clone <repo-url>
cd ai-writing-detector
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

### 2. Download the data

Download the dataset from [Kaggle](https://www.kaggle.com/datasets/navjotkaushal/human-vs-ai-generated-essays) and place the CSV at:

```
data/raw/AI_Human.csv
```

(The `data/` folder is gitignored so the dataset is never committed.)

### 3. Render the analysis

Each analysis lives in a Quarto document. To render one:

```bash
quarto render notebooks/01-eda.qmd
```

To render the whole project:

```bash
quarto render
```

## Modeling Plan (Summary)

- **Split:** stratified 80/20 train/test. The test set is held out until the very end.
- **Cross-validation:** stratified 5-fold CV on the training set for model selection and hyperparameter tuning, using `scikit-learn` pipelines so TF-IDF is fit only on the training fold (no leakage).
- **Models:** logistic regression and multinomial Naive Bayes as baselines, plus at least one stronger comparison model (linear SVM or gradient-boosted trees).
- **Metrics:** accuracy, precision / recall / F1 (per class and macro-averaged), ROC-AUC, and the confusion matrix. We report CV mean and standard deviation so that model differences can be judged against noise.
- **Error analysis:** inspect misclassified essays and the most informative TF-IDF features to check that the model is learning real linguistic patterns, not dataset artifacts.
